Iterative closest point (ICP)

The ICP (Iterative Closest Point) algorithm is a widely used technique in the field of computer vision and 3D reconstruction. Its main purpose is to align two or more point clouds by finding the optimal transformation that minimizes the distance between corresponding points in different clouds. The algorithm iteratively refines the transformation by iteratively matching and aligning the closest points between the clouds. Each iteration improves the alignment until convergence is reached. The ICP algorithm is highly effective in scenarios where initial pose estimation is known or can be estimated accurately. It has numerous applications in areas such as robotics, augmented reality, and medical imaging, enabling accurate registration and alignment of 3D data for further analysis and processing.

How was it done?

The first step involves importing the necessary packages and libraries to enable the implementation of the custom ICP algorithm. These packages provide the essential functionalities for working with point clouds and performing mathematical operations.

Following the importation of packages, a custom ICP function is defined. The input arguments for this function are the source and target point clouds. The source point cloud represents the initial state, while the target point cloud is the one that needs alignment with the source.

Within the ICP function, a homogeneous transformation matrix is defined. This matrix represents the initial transformation applied to the source point cloud. In the absence of a specific initial transformation, an identity matrix can be used, although it may result in slower convergence of the algorithm.

The alignment of point clouds is achieved by establishing corresponding points between the source and target clouds. The Open3D library provides a KDTree data structure, which facilitates efficient searching for the nearest neighbors. A KDTree is created for the source point cloud, and the correspondences of the target points in the source KDTree are identified. The order of this process is determined by the fact that the target cloud typically has fewer points and a different centroid compared to the source cloud.

Once the correspondences are established, the new de-meaned destination and source point clouds are computed. These de-meaned point clouds represent the centered versions of the original clouds. Singular Value Decomposition (SVD) is then applied to determine the rotation that best aligns the two point clouds.

Using the rotation obtained from SVD, the translation between the point clouds is calculated. This translation, along with the rotation, is used to derive a homogeneous transformation matrix. This matrix describes the overall transformation required for aligning the source and target clouds.

Finally, the obtained transformation matrix is applied to the source point cloud, resulting in its transformation to align with the target cloud. Additionally, the cost of the alignment is computed, providing an indication of the quality of the alignment. The cost can be calculated using either the correspondences or the de-meaned points, depending on the specific implementation.
